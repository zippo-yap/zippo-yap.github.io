Design to Web

"It’s supposed to be automatic, but actually you have to push this button." – John Brunner

Sunday, 3 April 2016

What is a responsive site, and why responsiveness is important?

A responsive site will be presented differently when you visit the same website on a different device e.g. mobile, desktop or tablet. This is important because of the different screen sizes of each device. For example, it solves the old problem of having to zoom in on a website when you're using your phone.

What is mobile first design, and why it's important?

Mobile devices are a a lot smaller than a desktop computer, so by designing a site to suit a mobile first perspective means when you move on to designing for larger screen sizes, you can add more content or elements to a site. It's easier to add elements to a larger site then having to cram content into a smaller screen.

What are frameworks, and their pros and cons?

A framework is a premade template that helps you build a website. All the elements like buttons, forms and headings have been preset for you so it makes getting a site up and running really quick. Most frameworks allow a developer to apply consistent styling throughout their site.

One of the downsides of using a framework is that it can be quite restrictive in terms of design features. They all differ so finding one that fits your needs as close as possible will make life easier and save you time having to write code to override preset styles.

What is a wireframe and why do we use it?

A wireframe is a really basic picture of what your site is going to look like. It's like the skeleton of a website and tells you how the content is going to be structured. When we use wireframes this allows us to concentrate more on the layout of information instead of fine details.

The aspects of my wireframe I found difficult to implement, and why.

I wanted to implement a menu that you could toggle from mobile/tablet view but I realised that you need javascript to be able to do this. I ended up coming up with a css version but it's still not quite right.

I really struggled with laying out my site how I wanted it and most of it was trial and error and a tonne of time spent fiddling with developer tools. I found working with the skeleton framwork quite tricky because I had to adjust so many different elements to make it suit the design I had in mind and I found it quite restrictive in terms of using the grid.

The contact form didn't make it to my final design because I figured I still need to know more before I can make a function contact form.
